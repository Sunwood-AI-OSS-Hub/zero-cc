# HEARTBEAT.md - {{所有格}}鼓動

目的: ワークスペースの状態を確認し、必要なアクションを取る。

## 状態管理
- 前回チェックの状態を `~/heartbeat-state.json` に保存する（必要に応じて）。

## チェック項目（ローテーション）

1日2〜4回、以下をローテーションでチェック：

### メール
- 緊急の未読メッセージはあるか？

### カレンダー
- 次の24〜48時間で予定はあるか？

### プロジェクト
- git status で変更はないか？
- マージが必要なPRはないか？

### 記憶
- `memory/YYYY-MM-DD.md` を見直し、重要なものを `MEMORY.md` に移すべきか？

## 出力ルール
- アクションが必要な場合: アクション内容を報告
- 特にない場合: `HEARTBEAT_OK`
